# 拼豆库存管理系统 - 运行说明

## 📋 运行步骤

### 第一步：安装依赖（如果还没安装）

打开 PowerShell 或命令提示符，在项目目录下运行：

```bash
pip install flask flask-cors
```

### 第二步：启动后端服务

在项目目录 `D:\pingdou` 下运行：

```bash
python app.py
```

看到类似这样的输出就说明启动成功了：
```
 * Running on all addresses (0.0.0.0)
 * Running on http://127.0.0.1:5000
 * Running on http://[你的IP地址]:5000
```

**注意**：这个窗口要保持打开，不要关闭！

### 第三步：打开前端页面

**请用浏览器访问后端地址（推荐）**

- 先按第二步启动 `python app.py`
- 浏览器打开：**`http://127.0.0.1:5000/`**  
  前端与接口同域，库存等接口才能正常加载。

**不要**再用「只双击打开 `index.html`（file 协议）」的方式，否则无法访问 `/api` 接口。

### 第四步：测试功能

1. **查看库存**：页面会自动加载 `data.json` 中的测试数据
2. **测试出库**：
   - 点击右上角的 "⊖ 手动出库" 按钮
   - 在弹窗中输入色号（如：H2）和数量（如：50）
   - 点击底部的 "手动出库" 按钮
   - 查看主页面的库存是否更新

## 🔧 常见问题

### 问题1：提示 "无法连接到本地服务"
- **原因**：后端服务没有启动
- **解决**：确保 `python app.py` 正在运行

### 问题2：提示 "ModuleNotFoundError: No module named 'flask'"
- **原因**：没有安装 Flask
- **解决**：运行 `pip install flask flask-cors`

### 问题3：端口 5000 被占用
- **原因**：5000 端口已被其他程序使用
- **解决**：修改 `app.py` 最后一行的 `port=5000` 为其他端口（如 5001）

## 📱 手机访问（局域网）

如果后端服务显示运行在 `http://[你的IP地址]:5000`，你可以：

1. 确保手机和电脑在同一个 WiFi 网络
2. 在手机浏览器访问：`http://[你的IP地址]:5000`（需要配置 Flask 提供静态文件服务）
3. 或者使用电脑的 IP 地址访问前端页面

## ☁️ GitHub + Render 部署（作品链接）

1. **安装 Git**（若命令行里没有 `git`）：从 https://git-scm.com/download/win 安装，重新打开终端。
2. **更新演示数据（可选）**：本地调好库存后，把 `data.json` 复制覆盖 `seed_data.json` 再提交，这样线上一打开就是你的模拟数据。
3. **推送到 GitHub**：在 `D:\pingdou` 下执行：
   ```bash
   git init
   git add app.py index.html seed_data.json requirements.txt runtime.txt render.yaml .gitignore 运行说明.md
   git commit -m "pingdou: Render deploy + seed data"
   git branch -M main
   git remote add origin https://github.com/你的用户名/pingdou.git
   git push -u origin main
   ```
4. **Render**：登录 https://render.com ，用 GitHub 授权，**New + → Blueprint** 选该仓库（会读 `render.yaml`），或 **Web Service** 手动填写：
   - **Build Command**：`pip install -r requirements.txt`
   - **Start Command**：`gunicorn app:app --bind 0.0.0.0:$PORT`
5. 部署完成后复制 **https://xxx.onrender.com** 填到美团「作品链接」。免费实例冷启动可能要等几十秒。

**说明**：`seed_data.json` 里的图纸若引用 `/uploads/xxx.webp`，需要你把对应图片放进仓库（例如自建 `seed_uploads` 并在启动时拷贝）才会显示缩略图；**豆子库存**不依赖这些文件，会按种子数据正常显示。
